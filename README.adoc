= Burble
:toc: preamble
:icons: font

Voice first. Friction last. Complexity optional.

== Overview

Burble is a modern, self-hostable, voice-first communications platform.
It combines Mumble-like voice seriousness with contemporary browser join flows,
strong permissions, excellent low-latency reliability, and privacy-respecting defaults.

The strategic aim is to become the **best serious voice system** with modern
onboarding, strong permissions, excellent reliability, and a future-proof
browser capability model.

== Architecture

[cols="1,3"]
|===
| Layer | Technology

| **Control plane**
| Elixir/Erlang (OTP) — auth, sessions, rooms, presence, permissions, signaling, moderation, audit

| **Media plane**
| WebRTC-compatible — RTP/SRTP, mixing/forwarding, recording, screen-share

| **Web client**
| ReScript + WebRTC APIs — first-class browser client

| **Desktop client**
| Tauri 2 — native wrapper around web client for power users

| **Protocol**
| Protobuf-defined wire protocol shared between server and all clients

| **Embedding**
| Client library for integration into IDApTIK (in-game voice) and PanLL (workspace voice)
|===

== Integration

Burble is a standalone product that also serves as an embeddable component:

* **IDApTIK** — in-game voice between Jessica and Q (asymmetric co-op multiplayer)
* **PanLL** — foundational voice layer for the panel workspace system

Client libraries expose a clean embedding API for these integrations.

== Quick Start

=== Prerequisites

* Elixir 1.17+ / Erlang/OTP 26+
* Zig 0.15+ (for coprocessor NIFs)
* Deno (for web client)
* VeriSimDB (database — run on port 8080)

=== Development Setup

[source,bash]
----
# 1. Start VeriSimDB
cd ../nextgen-databases/verisimdb && cargo run

# 2. Build Zig coprocessor NIFs
cd ffi/zig && zig build -Doptimize=ReleaseFast
cp zig-out/lib/libburble_coprocessor.so ../../server/priv/

# 3. Start Elixir server
cd server && mix deps.get && mix phx.server
# → http://localhost:4000

# 4. Start web client (separate terminal)
cd client/web && deno task dev
# → http://localhost:5173
----

=== Container Deployment (one command)

[source,bash]
----
cd containers
podman-compose -f compose.toml up
# → Server on :4000, Web on :8080, VeriSimDB on :8081
----

=== Run Tests

[source,bash]
----
cd server && mix test --no-start    # 49 tests, 0 failures
cd ffi/zig && zig build test        # Zig coprocessor unit tests
mix bench.coprocessor               # Elixir vs Zig performance benchmarks
----

== Repository Structure

[cols="1,3"]
|===
| Directory | Purpose

| `server/`
| Elixir/Phoenix control plane (OTP supervision tree)

| `media/`
| Media plane — WebRTC SFU, RTP handling, mixing

| `client/web/`
| ReScript web client (first-class browser experience)

| `client/desktop/`
| Tauri 2 desktop wrapper

| `client/lib/`
| Embeddable client library for IDApTIK and PanLL

| `protocol/`
| Protobuf wire protocol definitions

| `ffi/zig/`
| SIMD coprocessor NIFs (audio, DSP, neural, compression — 26,350x LZ4 speedup)

| `src/abi/`
| Idris2 ABI definitions (dependent type proofs for coprocessor interfaces)

| `containers/`
| Containerfiles + compose.toml (Chainguard images, Podman/Selur-compose)

| `.machine_readable/`
| SCM checkpoint files (STATE.scm, META.scm, ECOSYSTEM.scm)
|===

== Topology Modes

Burble supports four deployment topologies (set via `BURBLE_TOPOLOGY` env var):

[cols="1,1,1,1,1"]
|===
| Feature | Monarchic | Oligarchic | Distributed | Serverless

| **Default** | Yes | | |
| **Store** | VeriSimDB | Federated | Federated | None
| **E2EE** | Optional | Optional | Recommended | Mandatory
| **Recording** | Yes | Yes | Per-server | No
| **Moderation** | Full | Full | Per-server | None
| **Federation** | No | Cluster | Open | N/A
|===

== Coprocessor Performance

SIMD-accelerated audio processing via Zig NIFs:

[cols="1,1,1,1"]
|===
| Operation | Elixir | Zig | Speedup

| LZ4 compress | 83ms | 3.1µs | 26,350x
| Echo cancel | 19.4ms | 310µs | 62x
| FFT (256pt) | 826µs | 22µs | 37x
| Convolution | 300µs | 11µs | 27x
| Neural denoise | 54µs | 9µs | 6x
|===
